一、配置用户信息及常用操作

  1. cd 命令——改变目录
    • cd /e/useGit 进入E盘的 useGit 文件夹
    • cd ../返回上一级目录
    • pwd 打印当前目录

  1. 配置当前用户信息
//在全局下配置用户名和邮箱
git config --global user.name user.name 
git config --global user.e-mail user.e-mail
git config --list 查看所有配置，--list可简写为-l

  1. 创建文件、文件夹以及初始化
mkdir useGit 新建文件夹useGit
git init 初始化文件夹
ls 展示所有文件及文件夹列表，不包含隐藏文件
ls -a 展示所有文件及文件夹列表，包含隐藏文件
touch file.name 新建文件，可以创建隐藏文件（.xxx）
cat xxx.xxx 查看文件xxx.xxx中的内容
vi xxx.xxx //编辑xxx.xxx文件
    i  或者 a 插入信息//按i键或a键开始编辑
    Esc:wq 保存并退出
    Esc:q! 强制退出

二、git分区介绍及常用分区操作

  1. git分区介绍


  1. 常用分区操作

== 将文件从工作区添加到到暂存区，可同时添加多个文件
$ git add [file1] [file2] ...

==将文件夹从工作区添加到到暂存区
$ git add [dir]

==将当前目录下的所有文件从工作区添加到到暂存区
$ git add .

==把文件从暂存区添加到历史区
$ git commit -m "提交注释"

==联合提交，合并了add和commit
$ git commit -a -m "提交注释"

==查看文件状态（name是红色表示在工作区，绿色表示提交到了暂存区）
$ git status
git status命令可以列出当前目录所有还没有被git管理的文件和被git管理且被修改但还未提交(git commit)的文件

==查看当前分支的commit
$ git log

==查看整个本地仓储的commit
$ git reflog

==比较工作区(Workingspace)和暂存区(index)之间的差异,即修改后还没有暂存起来的变化内容
$ git diff

==工作区和历史区比较
$ git diff master
==暂存区和历史区比较
$ git diff --cached

==撤销，暂存区覆盖工作区，使两者一样
$ git checkout file.name

==回滚到上一个版本（三个区都一样）
$ git reset --hard id

==回退文件，将文件从暂存区回退到工作区
$ git reset HEAD file.name

三、创建分支与合并文件

  • $ git branch 查看分支
  • $ git branch branch.name 创建分支（默认创建的分支和创建之前的分支一模一样，创建的文件提交到某个分支上，这个文件才归属于这个分支）
  • $ git chekout branch.name切换分支
  • $ git branch -D branch.name删除分支（去别的分支上才可删除当前分支）
  • $ git checkout -b branch.name创建并切换分支
  • $ git merge branch.name在master上合并分支
    -$ git merge dev将其他分支中的内容合并到当前分支中，当两个分支上的同名文件内容不一致时，会发生冲突，可以修改文件，删除多余的，保留最终的结果，再用commit提交。

四、将本地仓库推送到远程仓库

下面以新建文件夹useGit，并将useGit推送到远程仓库为例。

  • 1 mkdir useGit新建一个文件夹，空文件夹不能被提交
  • 2 cd useGit
  • 3 $ git init初始化
  • 4touch README.MD 在文件夹中新建README.MD文件，它相当于一个项目说明书，空文件夹不能被推送到远程仓库
  • 5 echo ‘内容’ >> README.MD向README.MD文件中写入内容
  • 6 touch .gitignore增加一个忽略文件，在这个文件中写入的内容都不会被提交（常见的有.idea node_modules .DS_Store bower_components ）
  • 7 $ git add .
  • 8 $ git commit -m ''
  • 9 $ git remote add origin https://github.com/talktohebe/Doraemon.git 链接远程仓库
  • 10 $ git push origin master -u推送到远程仓库 （增加-u 以后不用指明 origin和master）

五、发布静态页

如果想让GitHub提供静态的网页，可以将内容发布到gh-pages分支上

  • 1 $ git checkout -b gh-pages新建并跳转到gh-pages分支
  • 2 $ git add .
  • 3 git commit -m '123'
  • 4 $ git push origin gh-pages
  • 5 在settings中找到新的地址预览网页，如果不是index.html需要手动添加地址
